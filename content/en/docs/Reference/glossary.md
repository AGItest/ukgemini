<h1>Glossary</h1>

<p><a name="A"></a><strong>acceptable quality level (AQL)</strong></p>
<p>threshold value applied to the results of testing data quality to determine whether the data meets criteria determined from a standard, specification or user requirements</p>
<p><strong>aggregated AQL</strong></p>
<p>acceptable quality level for aggregated results from a number of tests, e.g. 100% correct</p>
<p><a name="C"></a><strong>Coordinate Reference System</strong></p>
<p>a coordinate system that is related to an object by a datum [ISO 19111]</p>
<p><strong>CRS</strong></p>
<p>Coordinate Reference System</p>
<p><strong>CSW</strong></p>
<p>Catalogue Services for the Web. Http protocol request/respond binding for OGC Catalogue Service.</p>
<p>Note: see <a href="http://www.opengeospatial.org/standards/specifications/catalog">http://www.opengeospatial.org/standards/specifications/catalog</a></p>
<p><a id="D"></a><strong>Data Provider</strong></p>
<p>the organisation that creates the data and supplies the data for web publication, along with its metadata.</p>
<p>Note: for INSPIRE data, the Public Authority with the statutory obligation under the INSPIRE regulations.</p>
<p><strong>Data Publisher </strong></p>
<p>the organisation that publishes the data on the web and supplies data services to data users</p>
<p><strong>data resource</strong></p>
<p>dataset, collection of datasets or service supplying data</p>
<p><strong>dataset</strong></p>
<p>identifiable collection of data [ISO 19101]</p>
<p><strong>dataset series</strong></p>
<p>collection of datasets sharing the same specification, e.g. a 1:1250 scale map series [ISO 19115]</p>
<p><strong>discovery metadata</strong></p>
<p>information about a data or service resource, used to discover and access its suitability for sharing or re-use.</p>
<p><strong>DMS</strong></p>
<p>Discovery Metadata Service.</p>
<p>Note: one of a number of business services associated with the UK Location Information Infrastructure. Implemented by the CSW interface at <a href="http://csw.data.gov.uk/geonetwork/srv/en/main.home">http://csw.data.gov.uk/geonetwork/</a></p>
<p><strong>domain</strong></p>
<p>set of allowable values</p>
<p><strong>DTD</strong></p>
<p>Document Type Definition</p>
<p>Note: DTD is an XML schema language which defines the legal building blocks of an XML document. It is not relevant in the context of GEMINI2 metadata instances where <em>XSD</em> schemas are used to define the structure of XML documents.</p>
<p><strong><a id="E"></a>EDEN</strong></p>
<p>l’Equipe D’Experts en Normalisation</p>
<p>Note: a collaborative workspace for standardisation in the domain of geographic information. See <a href="http://eden.ign.fr/welcome">http://eden.ign.fr/welcome</a></p>
<p><strong>enumerated list</strong></p>
<p>set of values, each of which is identified by a code</p>
<p><strong>EPSG</strong></p>
<p>European Petroleum Survey Group (defunct but the abbreviation is retained in the name of the EPSG Geodetic Parameter Dataset). Since 2005 <em>OGP</em> Surveying and Positioning Committee.</p>
<p>Note: examples are given of the use of EPSG <em>URI</em>s in <em>metadata instances</em>. This does not imply that the EPSG registry is the <em>de jure</em> registry of geodetic parameters for UK Location.</p>
<p><strong><a id="G"></a>GEMET</strong></p>
<p>General Multilingual Environmental Thesaurus. </p>
<p>Note: developed by the European Topic Centre for the European Environment Agency. </p>
<p>Note: currently published and managed by the European Environmental Information and Observation Network.</p>
<p>Note: GEMET can be browsed and searched on-line, accessed through web services or downloaded as HTML or SKOS files, see <a href="http://www.eionet.europa.eu/gemet">http://www.eionet.europa.eu/gemet</a></p>
<p><strong>GEMINI2</strong></p>
<p>the UK discovery metadata profile of ISO 19115.</p>
<p>Note: in this document the term GEMINI2 means version 2.2 of UK GEMINI.</p>
<p><strong>geographic identifier</strong></p>
<p>label or code that identifies a location [adapted from ISO 19112]</p>
<p><strong>GML</strong></p>
<p>Geography Markup Language [ISO 19136]</p>
<p><strong>granularity</strong></p>
<p>resolution in terms of density or frequency</p>
<p><strong><a id="I"></a>INSPIRE</strong></p>
<p>infrastructure for Spatial Information in Europe</p>
<p><strong>ISO</strong></p>
<p>International Organisation for Standardisation</p>
<p><strong>ISO/TC 211</strong></p>
<p>The committee is responsible for standards such as ISO 19115, ISO 19136 and ISO 19139.</p>
<p>Notes: ISO Technical Committee 211 is responsible for standardisation in the field of digital geographic information. The home page of ISO / TC 211 is <a href="http://www.isotc211.org/">http://www.isotc211.org/</a></p>
<p><strong><a id="L"></a>location</strong></p>
<p>identifiable geographic place</p>
<p><strong><a id="M"></a>metadata</strong></p>
<p>data about data [ISO 19115]</p>
<p><strong>metadatabase</strong></p>
<p>collection of metadata about a set of data resources</p>
<p><strong>metadata element</strong></p>
<p>individual item of metadata relating to a data resource, e.g. Extent, Topic</p>
<p><strong>metadata instance</strong> </p>
<p>physically instantiated metadata.</p>
<p>Note: in the context of this document a metadata instance will be an <em>XML document</em> conforming to ISO / TS 19139 and other associated standards.</p>
<p><strong>metadata item</strong></p>
<p>a top level metadata concept in the UK GEMINI standard. Title, for example, is a metadata item.</p>
<p>Note: metadata items may comprise sub-items.</p>
<p><strong>metadata service</strong></p>
<p>service that supplies information about data resources</p>
<p><strong>metadataset</strong></p>
<p>identifiable set of metadata relating to a single data resource</p>
<p><strong><a id="O"></a>OGC</strong></p>
<p>Open Geospatial Consortium.</p>
<p>Note: an industry consortium of companies, government agencies and universities developing publicly available interface standards to geo-enable the Web, wireless, location-based services, and mainstream IT. See <a href="http://www.opengeospatial.org/">http://www.opengeospatial.org/</a></p>
<p><strong>OGP</strong></p>
<p>International Association of Oil and Gas Producers</p>
<p><strong><a id="P"></a>profile</strong></p>
<p>set of one or more base standards or subsets of base standards for a particular application</p>
<p><strong><a id="Q"></a>quality assessment</strong></p>
<p>review of quality of a data resource</p>
<p><strong>quality assurance</strong></p>
<p>process to ensure that quality is of an acceptable level</p>
<p><strong>quality control</strong></p>
<p>process of checking items to ensure that they are of an acceptable level of quality</p>
<p><strong>quality management</strong></p>
<p>overall process for assessing and controlling quality</p>
<p><strong>quality result</strong></p>
<p>value of a quality measure</p>
<p><strong><a id="S"></a>Schematron</strong></p>
<p>a language for making assertions about patterns in XML documents; ISO 19757 Document Schema Definition Language</p>
<p><strong>service</strong></p>
<p>application that provides information and /or functionality to other applications [adapted from Draft Implementing Rules for INSPIRE Transformation Services]</p>
<p><strong>SRS</strong></p>
<p>Spatial Reference System.</p>
<p>Note: a system for specifying position in the real world [ISO 19112]</p>
<p><strong><a id="U"></a>UK Information Infrastructure</strong></p>
<p>Infrastructure for the publication of location information as part of the implementation of the UK Location Strategy.  Incorporates the UK member state implementation of INSPIRE.</p>
<p><strong>UK Location Coordination Unit</strong></p>
<p>legal entity for the day-to-day coordination and management of the UK Location Information Infrastructure and the UK member state element of INSPIRE.</p>
<p><strong>UKLII</strong></p>
<p>UK Location Information Infrastructure</p>
<p><strong>universe of discourse</strong></p>
<p>view of the real or hypothetical world that includes everything of interest</p>
<p><strong>URI</strong></p>
<p>Uniform Resource Identifier.</p>
<p>Notes: URIs identify resources and may allow access to representations of the resources. A URL is a URI. An overview of W3C materials relating to address can be found at <a href="http://www.w3.org/Addressing/">http://www.w3.org/Addressing/</a></p>
<p><strong>URL</strong></p>
<p>Uniform Resource Locator. Specifies where an identified resource is available on the Web and the mechanism for retrieving it.</p>
<p><strong>UTC</strong></p>
<p>Coordinated Universal Time</p>
<p><strong>UUID (or GUID)</strong></p>
<p>Universally (or Global) Unique IDentifier (UUID) is a unique 128-bit integer that is represented as a 36 (or 32 ignoring the dashes) character string of hexadecimal numbers.</p>
<p>Note: UUIDs are system generated and ideally a UUID will never be generated twice by any computer in existence.</p>
<ul>
<li>Format: xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx</li>
<li>Example: 3ce4f380-b394-4e5d-b222-6914ea311156</li>
</ul>
<p><strong><a id="W"></a>WAF</strong></p>
<p>a Web Accessible Folder (WAF) is an HTTP accessible directory of files</p>
<p>Note: typically metadata files in XML format in which all files and their time-stamps are visible to a web browser or client. Crawlers are able to parse the file listings and date-time stamps and provide a search interface on these documents.</p>
<p><strong>WMS</strong></p>
<p>Web Mapping Service</p>
<p><strong>WFS</strong></p>
<p>Web Feature Service</p>
<p><strong><a id="X"></a>XLink</strong></p>
<p>XML Linking Language</p>
<p>Note: used to include metadata element data by-reference rather than by-value.</p>
<p>Note: the XLink specification can be found at <a href="http://www.w3.org/TR/xlink/">http://www.w3.org/TR/xlink/</a></p>
<p><strong>XML</strong></p>
<p>eXtensible Markup Language</p>
<p>Note: the XML specification can be found at <a href="http://www.w3.org/TR/REC-xml/">http://www.w3.org/TR/REC-xml/</a></p>
<p><strong>XML Document</strong></p>
<p>a collection of data represented in XML</p>
<p><strong>XML parser </strong></p>
<p>software component responsible for reading an XML document and dividing it into individual elements and attributes.</p>
<p>Note: parsers will check that an XML document is well-formed. Some parsers will undertake a schema validation.</p>
<p><strong>XSD</strong></p>
<p>XML Schema Definition language.</p>
<p>Note: an XSD is a document written in XML that defines the structure of an XML document.</p>
